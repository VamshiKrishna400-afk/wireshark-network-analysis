# Volume-Based DoS Attacks

## 1. ICMP Flood Attack

### Objective

An ICMP Flood Attack is a Denial-of-Service (DoS) technique used to overwhelm a target system by sending a large volume of ICMP Echo Request packets. The excessive traffic can consume network bandwidth and system resources, potentially degrading performance or causing service disruption.

### Attack Tool

**hping3**

### Command Used

```bash id="icmpflood"
sudo hping3 --icmp --flood <target-ip>
```

### Command Explanation

| Option        | Description                                                    |
| ------------- | -------------------------------------------------------------- |
| `sudo`        | Runs hping3 with administrative privileges.                    |
| `--icmp`      | Sends ICMP Echo Request packets.                               |
| `--flood`     | Sends packets as fast as possible without waiting for replies. |
| `<target-ip>` | IP address of the target host.                                 |

### Working

1. hping3 continuously generates ICMP Echo Request packets.
2. Packets are transmitted at a very high rate toward the target.
3. The target attempts to process and respond to the requests.
4. Excessive traffic may consume bandwidth and processing resources.
5. Network performance may degrade depending on the attack volume and target capacity.

### Attack Execution Screenshot

![ICMP Flood Attack](screenshots/attacker/icmp_flood_attack.png)

### Wireshark Display Filter

```wireshark

```

### Filter Explanation

Displays ICMP packets generated during the attack, including:

* ICMP Echo Request
* ICMP Echo Reply (if generated by the target)

This filter helps isolate and analyze the flood traffic within the packet capture.

### Wireshark Analysis Screenshot

![ICMP Flood Attack Analysis](screenshots/wireshark/icmp_flood_capture.png)

### Packet Capture

[Wireshark Pcap file](pcaps/icmp_flood.pcap)

### Observations

* A high volume of ICMP Echo Request packets was observed.
* Packets were transmitted continuously without waiting for responses.
* Network traffic increased significantly during the attack.
* The target generated ICMP Echo Replies when resources permitted.
* Wireshark clearly displayed the abnormal packet rate.

### Security Significance

ICMP Flood attacks are commonly used to exhaust network bandwidth and system resources, leading to degraded performance or denial of service. Monitoring abnormal spikes in ICMP traffic can help detect and mitigate such attacks.

## 3. TCP Flood Attack

### Objective

A TCP SYN Flood Attack is a Denial-of-Service (DoS) technique used to overwhelm a target system by sending a massive number of TCP SYN packets. The attack exploits the TCP three-way handshake process by creating numerous half-open connections, consuming server resources and potentially preventing legitimate connections.

### Attack Tool

**hping3**

### Command Used

```bash id="tcpsynflood"
sudo hping3 -S --flood -p 80 <target-ip>
```

### Command Explanation

| Option        | Description                                                    |
| ------------- | -------------------------------------------------------------- |
| `sudo`        | Runs hping3 with administrative privileges.                    |
| `-S`          | Sets the TCP SYN flag in transmitted packets.                  |
| `--flood`     | Sends packets as fast as possible without waiting for replies. |
| `-p 80`       | Targets TCP port 80 (HTTP service).                            |
| `<target-ip>` | IP address of the target host.                                 |

### Working

1. hping3 continuously generates TCP SYN packets.
2. Packets are transmitted at a very high rate to TCP port 80.
3. The target responds with SYN-ACK packets and allocates resources for each connection request.
4. The final ACK required to complete the TCP handshake is never sent.
5. A large number of half-open connections accumulate, consuming system resources.

```wireshark id="tcpsynfloodfilter"

```

### Filter Explanation

Displays TCP packets generated during the attack, including:

* TCP SYN Packets
* TCP SYN-ACK Responses

This filter helps isolate and analyze SYN flood traffic and connection attempts.

### Wireshark Analysis Screenshot

![TCP Flood Attack Analysis](screenshots/wireshark/tcp_flood_capture.png)

### Packet Capture

[Wireshark Pcap file](pcaps/tcp_flood.pcap)

### Observations

* A high volume of TCP SYN packets was transmitted to port 80.
* The target generated numerous SYN-ACK responses.
* Many TCP connections remained half-open.
* Network traffic and connection attempts increased significantly.
* Wireshark clearly displayed the abnormal SYN packet rate.

### Security Significance

TCP SYN Flood attacks are among the most common DoS techniques used to exhaust server resources and disrupt services. Monitoring abnormal SYN traffic patterns, implementing SYN cookies, and using rate-limiting mechanisms can help detect and mitigate such attacks.

# Protocol-Based DoS Attacks

## 1. TCP SYN Flood Attack

### Objective

A TCP SYN Flood is a protocol-based Denial-of-Service (DoS) attack that exploits the TCP three-way handshake mechanism. The attack aims to exhaust the target's connection resources by creating a large number of half-open TCP connections, preventing legitimate users from accessing the service.

### Attack Tool

**hping3**

### Command Used

```bash id="tcpsynflooddos"
sudo hping3 -S --flood -p 80 <target-ip>
```

### Command Explanation

| Option        | Description                                  |
| ------------- | -------------------------------------------- |
| `sudo`        | Runs hping3 with administrative privileges.  |
| `-S`          | Sets the TCP SYN flag.                       |
| `--flood`     | Sends packets continuously at maximum speed. |
| `-p 80`       | Targets TCP port 80 (HTTP service).          |
| `<target-ip>` | IP address of the target host.               |

### Working

1. The attacker continuously sends TCP SYN packets to the target.
2. The target responds with TCP SYN-ACK packets and allocates memory for each connection request.
3. The attacker does not complete the TCP three-way handshake by sending the final ACK.
4. Thousands of half-open connections accumulate on the target.
5. Server resources become exhausted, reducing the ability to process legitimate requests.

### Attack Execution Screenshot

![TCP SYN Flood Attack](screenshots/attacker/tcp_syn_flood_attack.png)

### Wireshark Display Filter

```wireshark
tcp.flags.syn == 1
```

### Filter Explanation

Displays TCP packets containing the SYN flag, including:

* TCP SYN Requests
* TCP SYN-ACK Responses

This filter helps identify excessive connection requests characteristic of a SYN Flood attack.

### Wireshark Analysis Screenshot

![TCP SYN Flood Analysis](screenshots/wireshark/tcp_syn_flood_capture.png)

### Packet Capture

[Wireshark Pcap file](pcaps/tcp_syn_flood.pcap)

### Observations

* A large number of TCP SYN packets were observed.
* The target generated continuous SYN-ACK responses.
* Numerous half-open TCP connections were created.
* Network traffic volume increased significantly.
* Legitimate connection requests may experience delays or failures.

### Security Significance

TCP SYN Flood attacks exploit a fundamental weakness in the TCP connection establishment process. They are widely used to disrupt web servers, application servers, and network services by exhausting connection tables and system resources. Detection typically involves monitoring abnormal spikes in SYN traffic and excessive half-open connections.

## 2. TCP ACK Flood 

### Objective

A TCP ACK Flood is a protocol-based Denial-of-Service (DoS) attack that overwhelms a target system or network device with a high volume of TCP ACK packets. The attack aims to consume processing resources on servers, firewalls, intrusion detection systems, and load balancers by forcing them to inspect and process excessive ACK traffic.

### Attack Tool

**hping3**

### Command Used

```bash id="tcpackflooddos"
sudo hping3 -A --flood -p 80 <target-ip>
```

### Command Explanation

| Option        | Description                                  |
| ------------- | -------------------------------------------- |
| `sudo`        | Runs hping3 with administrative privileges.  |
| `-A`          | Sets the TCP ACK flag.                       |
| `--flood`     | Sends packets continuously at maximum speed. |
| `-p 80`       | Targets TCP port 80 (HTTP service).          |
| `<target-ip>` | IP address of the target host.               |

### Working

1. The attacker continuously sends TCP ACK packets to the target.
2. The target, firewall, or IDS/IPS processes each incoming ACK packet.
3. Since the ACK packets do not belong to valid TCP sessions, the target may generate TCP RST responses or simply process and discard them.
4. Large volumes of ACK packets consume CPU, memory, and network resources.
5. Excessive traffic can degrade service performance and impact legitimate users.

## 6. LAND Attack 

### Objective

A LAND Attack is a protocol-based Denial-of-Service (DoS) attack that sends spoofed TCP packets with the source IP address set to the same value as the destination IP address. The attack attempts to confuse the target system into responding to itself, potentially consuming resources and disrupting normal network operations.

### Attack Tool

**hping3**

### Command Used

```bash id="landattack"
sudo hping3 -S -a <target-ip> <target-ip>
```

### Command Explanation

| Option           | Description                                              |
| ---------------- | -------------------------------------------------------- |
| `sudo`           | Runs hping3 with administrative privileges.              |
| `-S`             | Sets the TCP SYN flag.                                   |
| `-a <target-ip>` | Spoofs the source IP address as the target's IP address. |
| `<target-ip>`    | IP address of the target host.                           |

### Working

1. The attacker sends TCP SYN packets to the target.
2. The source IP address is spoofed to match the target's own IP address.
3. The target receives packets that appear to originate from itself.
4. The target attempts to process and respond to the spoofed requests.
5. Excessive spoofed traffic can consume resources and disrupt normal operations.

### Attack Execution Screenshot

![LAND Attack](screenshots/attacker/land_attack.png)

### Wireshark Display Filter

### Filter Explanation

Displays spoofed TCP SYN packets where the source and destination IP addresses are identical.

This filter helps identify LAND attack traffic and analyze self-referential packet behavior.

### Packet Capture

### Security Significance

LAND attacks exploit IP spoofing and protocol handling mechanisms to generate abnormal traffic patterns. While modern operating systems are generally resistant to classic LAND attacks, the technique remains valuable for studying spoofed packet behavior, protocol abuse, and DoS attack detection in network monitoring environments.

## 7. Fragmentation Attack

### Objective

A Fragmentation Attack is a protocol-based Denial-of-Service (DoS) attack that overwhelms a target system with a large volume of fragmented IP packets. The attack aims to consume CPU, memory, and packet reassembly resources by forcing the target to reconstruct numerous fragmented packets before processing them.

### Attack Tool

**hping3**

### Command Used

```bash id="fragmentationattack"
sudo hping3 -S -f --flood -p 80 <target-ip>
```

### Command Explanation

| Option        | Description                                  |
| ------------- | -------------------------------------------- |
| `sudo`        | Runs hping3 with administrative privileges.  |
| `-S`          | Sets the TCP SYN flag.                       |
| `-f`          | Fragments the outgoing IP packets.           |
| `--flood`     | Sends packets continuously at maximum speed. |
| `-p 80`       | Targets TCP port 80 (HTTP service).          |
| `<target-ip>` | IP address of the target host.               |

### Working

1. The attacker generates TCP SYN packets.
2. Each packet is fragmented at the IP layer before transmission.
3. The target receives a large number of fragmented packets.
4. The operating system attempts to reassemble the fragments into complete packets.
5. Excessive reassembly operations consume system resources and may degrade performance.

Fragmentation attacks exploit the packet reassembly process of network devices and operating systems.
